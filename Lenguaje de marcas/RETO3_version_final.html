Contexto del Proyecto

Somos Kbyte SL, una empresa de software que surge en 1985 y hoy en día tiene un equipo de más de 2000 personas que operan en 10 países distintos. Ayudamos a diferentes compañías del ámbito privado y público a ampliar sus horizontes.
El trabajo de calidad y la confianza de nuestros clientes ha hecho que nuestra empresa crezca y mejore con los años.
Hemos sido contratados por la empresa Rolls-Royce ya que ha creado un departamento nuevo el cual se encarga de fabricar modelos de una gama menor y más comercial a lo que están acostumbrados y quieren nuestra ayuda para solventar ciertos problemas.

Sobre Rolls-Royce

Rolls-Royce es conocida mundialmente por fabricar coches de lujo de alta gama. Sin embargo, con este nuevo departamento buscan entrar en un mercado más accesible sin perder su esencia de calidad y prestigio.

Nuestro Rol

Como empresa de software, vamos a desarrollar soluciones tecnológicas que faciliten la gestión, producción y comercialización de esta nueva gama de vehículos.

Problemas Identificados

Problema 1: Gestión de Inventario

El nuevo departamento necesita un sistema para controlar las piezas y componentes de los modelos más comerciales.

Problema 2: Control de Producción

Falta un sistema que permita seguir el estado de cada vehículo durante el proceso de fabricación.

Problema 3: Gestión de Clientes

Necesitan una base de datos para gestionar los pedidos y preferencias de los clientes de la nueva gama.

Problema 4: Posibles dudas y quejas

Necesitan la simulación o prototipo de un formulario para consultas, una plantilla de reclamaciones y un FAQ

Módulos del Proyecto

El proyecto lo hemos dividido en los siguientes apartados, que son los módulos que cursamos:

Módulo 1: Bases de Datos

La base de datos presentada está diseñada para gestionar el proceso de fabricación, montaje y control de calidad de coches dentro de una fábrica. Su estructura permite relacionar coches, piezas, empleados, puestos de trabajo, tipos de pruebas de calidad y estados del vehículo, garantizando una organización clara y eficiente de la información.

Módulo 2: Programación

Desarrollaremos un programa en lenguaje Python para la gestión de inventario pudiendo: almacenar, añadir, modificar y eliminar componentes y sus cantidades.
El programa lo haremos en colab permitiendo a nuestros clientes acceder a la base de datos provisional (SQLite) desde cualquier sucursal o localización para poder gestionar el inventario.

Módulo 3: Entornos de desarrollo

Crearemos un repositorio en Github en el que trabajaremos toodos los integrantes del grupo, permitiendonos llevar un control de versiones.
También procuraremos documentar tanto el programa de python como el HTML (Añadiendo comentarios para dar explicaciones utiles).

Módulo 4: Lenguaje de Marcas

Crearemos interfaces web intuitivas para que los empleados puedan usar el sistema fácilmente.
En la web Se podrá ver que problemas tienen nuestros clientes y nuestras soluciones, como una animación simulando la línea de producción y donde esta el cuello de botella.
Aparecera una tabla explicando el tiempo que se le ha dedicado a cada apartado y una conclusión. A cada uno de estos apartados podrás navegar, desde el navegador que está en la parte superior de la pagina web.
Finalmente, en la solución de el problema 4 hemos añadido un botón que nos llevará a un apartado en el que hemos cumplido con las exigencias del módulo de inglés.

Conclusión

Con esta propuesta esperamos solucionar los problemas que presenta Rolls-Royce en su nuevo departamento y ayudarles a mejorar su gama comercial.

Beneficios Esperados

  • Mayor control sobre el inventario
  • Reducción de errores en el proceso de fabricación, con la animación
  • Establecer un mecanismo de comunicación eficiente, con el formulario y la plantilla de reclamaciones
  • Sistema escalable para futuro crecimiento

Estamos comprometidos en entregar un proyecto de calidad que cumpla con todas las expectativas de Rolls-Royce.
